#### 简单工厂

通过工厂获取一个接口对象，调用接口的方法实现所需要的功能

```
提供一个创建对象实例的功能,不需要关心其具体实现,被创建实例的类型可以是interface,abstract class,class
(非标准的设计模式,经典)
```

##### 1.场景问题

```
接口回顾:
	接口特殊抽象类,所有field都是constant,所有method都是abstract,只定义method,没有method的implement
    接口定义类的行为,具体的实现类实现接口约定的功能外,可按需实现其他功能
    不同的类可根据接口实现相同的行为
    接口的思想即封装隔离,
    	封装指对被隔离体的行为(功能)的封装,
        隔离指外部调用与内部实现,外部调用即只能通过接口进行调用,外部调用与内部实现通过接口隔开
    接口vs抽象类:
         优先用接口
         既要定义子类的行为,又要为子类提供公共的功能时选抽象类
    面向接口编程
        三层架构,层与层之间通过接口进行通信,每一层有多个小模块,对外提供一个接口,通过接口调用某个模块的功能
    不用设计模式的解决方案及问题
    	既要知道接口又要知道具体的实现类
```

##### 2.简单工厂模式

```
提供创建对象实例的功能,无需关心其具体实现,被创建的对象可以是接口,抽象类,具体类
```

  ![](.\img\1.png)

```
1.SimpleFactory可用来创建接口,也可创建抽象类甚至是普通类的实例
2.SimpleFactory的方法是静态的,又称静态工厂,Factory构造方法私有化
3.SimpleFactory可以有多个构造方法,用于创建不同的接口,抽象类,类实例  
4.命名:
	类名称:模块名称+Factory  用户模块UserFactory
	方法名称:get+接口名称 create+接口名称  getUserEbi/createUserEbi
5.SimpleFactory的方法就是选择合适的实现类来创建实例对象
	根据客户端的参数/配置文件中的参数/程序运行期的某个值(缓存中的某个运行期的值)
6.SimpleFactory根据传入的参数创建对象,要求客户端理解参数对应的功能,一定程度上向客户暴露内部实现细节
7.不修改工厂类增加新的实现类:
	去配置文件中读取全限定路径名创建实例对象
8.SimpleFactory
	实现封装,面向接口编程
	解耦:客户端与具体实现类的解耦  客户端通过工厂获取指定接口对象
	客户端需了解每个参数的含义,定制化实例对象,一定程度上暴露内部实现
	构造方法私有化,静态方法创建接口  无法创建子类来重写接口实现具体功能
9.本质:选择某个实现类  重在选择
10.场景:通过工厂获取相应的接口无需关心具体的实现
11.简单工厂vs其他模式
	简单工厂是选择实现的,重在选择,可以有多个用于选择并创建对象的方法,多个方法创建的对象之间有无关系均可
	抽象工厂模式是选择产品簇的实现的,内部有多个用于选择并创建对象的方法,但此方法所创造的对象之间是有联系的,是构成一个产品簇所需要的部件对象,抽象工厂退化为只有一个实现,不分层次,相当于简单工厂
	工厂方法本质也是选择实现,工厂方法将选择具体实现的功能延迟到子类去实现,若将工厂方法中选择的实现放到父类中直接实现即视为简单工厂
	简单工程的本质是选择实现,可跟任何其他创建对象实例模式搭配使用eg:单例模式,原型模式,生成器模式
```
